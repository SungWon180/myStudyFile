## 1-1. data?

> 관찰의 결과로 나타난 정량적 혹은 정성적인 실제 값.
> <br>
> ex. 에베레스트의 높이 : 8848m

## 1-2. Information?

> 데이터를 기반으로 하여 의미를 부여한 것.
> <br>
> ex. 에베레스트는 세계에서 가장 높은 산이다.

## 1-3. Database?

> 한 조직에 필요한 정보를 여러 응용 시스템에서 공용할 수 있도록 논리적으로 연관된 데이터를 모으고, 중복되는 데이터를 최소화하여 구체적으로 통합/저장해 놓은 것.
> 구조화된 데이터의 집합이므로 컴퓨터에서 쉽게 접근, 관리, 업데이트할 수 있다.
> <br> <span style="color:yellow;">※ 중복되는 데이터 최소화 > 데이터베이스 무결성</span>

<details>
  <summary>정의</summary>

1. 운용 데이터 : 조직의 목적을 위해 사용되는 데이터
2. 공용 데이터 : 공통으로 사용되는 데이터
3. 통합 데이터 : 중복을 최소화하여 중복으로 인한 데이터 불일치 현상 제거
4. 저장 데이터 : 컴퓨터 저장 장치에 저장된 데이터
</details>
<br>
<details>
  <summary>특징</summary>

5. 실시간 접근성 : 사용자가 데이터를 요청하면 실시간으로 결과를 서비스한다.
6. 계속적인 변화 : 데이터 값은 시간에 따라 항상 바뀐다.
7. 동시 공유 : 데이터베이스는 서로 다른 업무 또는 여러 사용자에게 동시 공유된다.
8. 내용에 따른 참조 : 데이터베이스에 저장 된 데이터는 데이터의 물리적 위치가 아니라 데이터 값에 따라 참조된다.
</details>

## 2-1. DBMS?

> 데이터베이스에서 <span style="color:yellow;">데이터를 추출, 조작, 정의, 제어</span> 등을 할 수 있게 해주는 데이터베이스 전용 관리 프로그램

<details>
  <summary>기능</summary>

1. 데이터 추출 : 사용자가 조회하는 데이터 혹은 응용 프로그램의 데이터를 추출 (READ)
2. 데이터 조작 : 데이터를 조작하는 소프트웨어가 요청하는 데이터의 삽입, 수정, 삭제 작업을 지원 (C,U,D 기능 / R제외)
3. 데이터 정의 : 데이터의 구조를 정의하고 데이터 구조에 대한 삭제 및 변경 기능을 수행
4. 데이터 제어 : 데이터가 소실됐는데 복구 불가능하거나 데이터끼리 매치가 되지 않거나 문제 생길시 백업 및 회복, 동시성 제어 등등의 그능을 지원
</details>
